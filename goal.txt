Build a full-stack web application with the following requirements.

Tech Stack:
- Frontend: React.js + Tailwind CSS
- Backend: Node.js + Express.js
- Database: MongoDB (with Mongoose)
- Authentication: JWT + OAuth (Google & GitHub)
- File Storage: Local storage or cloud (like AWS S3 optional)
- PDF Rendering: Use a React PDF viewer library

Core Features:

1. Authentication System:
   - Users can sign up with:
     - Email + Password (with email verification via OTP or link)
     - OAuth login via Google and GitHub
   - Users can log in using:
     - Email or username + password
   - Passwords must be hashed using bcrypt
   - Use JWT for session management
   - Store user info: username, email, password, OAuth IDs, verified status

2. User Dashboard:
   - After login, redirect to dashboard
   - Display:
     - Books uploaded by the user
     - Books uploaded by other users (public library)
   - Provide filtering/search functionality

3. Book Management:
   - Users can:
     - Upload a book (PDF)
     - Delete their uploaded books
   - Store metadata:
     - Title
     - Author
     - Description
     - Uploaded by (user reference)
     - Upload date

4. Book Reader:
   - Users can open and read PDFs in-browser
   - Include features:
     - Page navigation
     - Zoom in/out
     - Dark mode
     - Bookmark pages
     - Highlight text (basic)
   
5. Reading Progress Tracking:
   - Track:
     - Last page read
     - Percentage completed
   - Save progress per user per book in database
   - Resume reading from last position

6. UI/UX:
   - Use Tailwind CSS for modern responsive design
   - Clean dashboard layout with sidebar/navigation
   - Smooth transitions and loading states
   - Mobile responsive

7. Backend API Structure:
   - Auth routes:
     - /api/auth/register
     - /api/auth/login
     - /api/auth/google
     - /api/auth/github
     - /api/auth/verify-email
   - Book routes:
     - /api/books/upload
     - /api/books/delete/:id
     - /api/books/all
     - /api/books/user
   - Progress routes:
     - /api/progress/save
     - /api/progress/get/:bookId

8. Database Models (Mongoose):
   - User Schema
   - Book Schema
   - Progress Schema

9. Security:
   - Validate all inputs
   - Protect routes using middleware
   - Handle file uploads securely (limit size, type PDF only)

10. Project Structure:
   - /client → React frontend
   - /server → Express backend
   - Separate routes, controllers, models, middleware

11. Extra Features (Optional but preferred):
   - Like/Favorite books
   - Comments/reviews
   - Recently read section
   - Pagination for books
    
Deliverables:
- Clean, modular code
- Comments explaining key logic
- Setup instructions (README)
- Environment variable usage (.env)

Start by:
1. Creating backend structure and authentication system
2. Then build frontend pages (Login, Register, Dashboard)
3. Then integrate book upload and reader
4. Finally implement progress tracking
